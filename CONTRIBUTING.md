Contributing to OpenTrack

Hi!
Thank you for your interest in contributing to this project.

This project was originally built as part of an academic assignment, but contributions are welcome to improve it further.

---

~How to Contribute

You can contribute in the following ways:

- Fix bugs or errors
- Improve code readability
- Enhance data validation
- Improve file handling or reporting
- Improve documentation

---

~Steps to Contribute

1. Fork the repository

2. Clone your fork
   
   git clone https://github.com/your-username/OpenTrack.git

3. Create a new branch
   
   git checkout -b feature-name

4. Make your changes

5. Commit your changes
   
   git commit -m "Added feature / Fixed bug"

6. Push to your branch
   
   git push origin feature-name

7. Open a Pull Request

---

~Guidelines

- Keep code simple and readable
- Follow basic Python practices
- Do not use advanced libraries (this project focuses on fundamentals)
- Add comments where necessary
- Make sure the code runs without errors from top to bottom

---

~Important Notes

- This project uses only basic Python concepts
- Avoid adding complex frameworks or external libraries
- Keep changes beginner-friendly

---

~Suggestions

If you have ideas to improve the project:

- Better validation
- Improved reporting format
- More analytics features

Feel free to open an issue or suggestion!

---

Thanks again for contributing! 🚀
